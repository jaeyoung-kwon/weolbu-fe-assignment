# 월부 FE 과제 요구사항 정리

## 0. 목표/범위

- 강의 등록 및 신청에 필요한 UI를 구현한다.
  - 회원 가입
  - 강의 개설(강의 등록)
  - 수강 신청(강의 목록 + 다중 선택 신청)
- 모바일 웹 기준으로 구현하며, UI는 와이어프레임을 참고하되 자유롭게 구성한다.
- 백엔드 API는 제공된 Docker 이미지를 다운로드/실행하여 연동한다.

## 1. 기능 요구사항

### 1.1 회원 가입

- [ ] 이름/이메일/휴대폰 번호/비밀번호/회원 유형(수강생/강사)을 입력받는다.
- [ ] "가입하기" 클릭 시 회원 가입이 완료된다.
- [ ] 가입 완료 후 강의 목록 화면으로 이동한다.
- [ ] 비밀번호 검증
  - [ ] 6자 이상 10자 이하
  - [ ] 영문 소문자/대문자/숫자 중 최소 2종 조합

### 1.2 강의 개설(강의 등록)

- [ ] 강의 목록에서 "강의 개설" 버튼을 통해 진입한다.
- [ ] 강사 회원만 강의를 등록할 수 있다. (수강생은 등록 불가)
- [ ] 강의 등록 입력 항목
  - [ ] 강의명
  - [ ] 최대 수강 인원
  - [ ] 가격
- [ ] "등록하기" 클릭 시 등록 완료되며 강의 목록으로 돌아간다.

### 1.3 수강 신청(강의 목록)

- [ ] 강의 목록은 무한 스크롤로 하위 아이템이 자연스럽게 로드되어야 한다.
- [ ] 정렬 기능(3종)
  - [ ] 최근 등록순
  - [ ] 신청자 많은 순(현재 신청자 수 내림차순)
  - [ ] 신청률 높은 순(신청률 = 신청자 수 / 최대 수강 인원, 내림차순)
- [ ] 강의 아이템에 아래 정보를 표시한다.
  - [ ] 강의명
  - [ ] 가격
  - [ ] 강사명
  - [ ] 현재 신청자 수
  - [ ] 최대 수강 인원수
- [ ] 강의는 여러 개를 선택해서 한 번에 신청할 수 있다.
- [ ] 강사 회원도 수강 신청이 가능하다.

## 2. 개발 조건

- [ ] React(18+) 또는 Vue(3+) 중 1개 선택
- [ ] TypeScript만 사용
- [ ] 검증된 패키지 자유 사용

## 3. 우대 조건(가산 요소)

- [ ] Next(App Router, 13+) 또는 Nuxt(3+) 적용
- [ ] SSR 구현
- [ ] Semantic HTML 준수
- [ ] Flexbox/Grid 레이아웃 구성
- [ ] 로직/컴포넌트 테스트 코드 작성
- [ ] README에 프로젝트 간단 설명 작성
- [ ] 스펙 외 추가 기능 구현(있으면 좋을 기능) — 가산점

## 4. 주의사항

- 사용해보지 않은 최신 기술 학습보다, 본인에게 가장 효율적인 방식으로 요구사항을 적시에 구현하는 것을 우선한다.
